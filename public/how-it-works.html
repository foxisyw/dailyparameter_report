Detailed Process Flow
Phase 1 Cron Trigger & EMA Data Collection ~15 min
GitHub Actions
Cron Job Triggered Automated
GitHub Actions fires on schedule (09:00 HKT / 13:00 HKT) or manual dispatch
schedule: cron '0 1 * * *' & '0 5 * * *'
Runner
Start Realtime WebSocket Server
Connects to OKX WebSocket API to collect live market data: bid/ask, index price, buy/sell limits
ws://www.okx.com/ws/v5/public → polling every 5s
Realtime Server
Calculate 24h EMA Indicators
For each instrument, computes exponential moving averages (tau=24h):
basis
Premium/discount vs index
spread
Bid-ask spread ratio
limitUp_buffer
Distance to upper limit
limitDn_buffer
Distance to lower limit
EMA Collector
Poll Coverage → Stop When >80%
Polls /health every 30s. Once 80%+ instruments have EMA data, kills the server and saves cache.
ema_state.json → 1488 instruments cached
Phase 2 OKX API Data Fetch (Parallel) ~2 min
Price Limit Adapter
Fetch All Live Instruments
Three parallel requests for SPOT, SWAP, FUTURES instrument lists
GET /api/v5/public/instruments?instType={SPOT|SWAP|FUTURES}
Price Limit Adapter
Fetch XYZ Cap Parameters (Batched)
For each 50-instrument batch, fetch Y caps (inner band) and Z caps (outer hard limit)
GET /priapi/v5/public/products?instType={type}&instId={ids}&includeType=1
~30 batches → 1472 instruments with lpY1/lpY2/lpZ1/lpZ2
Asset Map
Load Asset Classification
Maps each base currency → asset type from assets_types.md
BTC → Topcoins, AAPL → TradFi, USDT → Fiat, PEPE → Altcoins
Phase 3 Review Rules Engine (4 Rules) < 1 min
Rule Engine
Run 4 Compliance Rules Against All Instruments
Rule 1: Buffer Too Tight
If limitUp/Dn buffer EMA < 0 → price is persistently near a limit. Compare B/A spread with Y cap spread. If too tight → widen Y cap.
Rule 2: Asymmetric Basis
Compare basis EMA with Z cap. If basis is large relative to the relevant Z cap → widen Z cap to accommodate premium/discount.
Rule 3: Asset-Type Consistency
Check Y/Z caps match expected ranges for the asset type: Topcoins Y≈1%, Z≈2%; Altcoins Y≤4%, Z≤10%; Fiat Y≈1%, Z≈2%.
Rule 4: Z Cap > Y Cap
Z cap (outer band) must always be wider than Y cap (inner band). If not → misconfiguration, needs immediate fix.
Rule Engine
Generate Findings & Recommended Changes
Each rule produces a list of flagged instruments with issue description, current values, expected values, and suggested fix.
Decision Logic: For each flagged instrument, calculate the adjustment based on asset-type defaults and current EMA data. Generate CSV files in OKX's expected upload format (spot template vs perp template).
